# DMAIC Roadmap

## Definition

DMAIC is a data-driven, problem-solving methodology structured into five sequential phases: Define (map the process and understand customer requirements), Measure (collect baseline data and assess current performance), Analyze (identify waste types and root causes), Improve (design future state, test solutions, and validate), and Control (create control systems and spread improvements). Each phase has specific tools, activities, and deliverables. The methodology provides a scalable roadmap that Lean and Six Sigma tools plug into—Lean tools (Value Stream Mapping, 5-S) for flow and waste elimination; Six Sigma tools (statistical analysis, capability analysis) for variation and defect reduction.

## In the Book

Slides 61-62 define the DMAIC Roadmap as "a data-driven problem solving methodology with a structured Roadmap (D-M-A-I-C)." The workbook details each phase:

- **DEFINE:** Map and measure the process; determine customer requirements. Deliverables: Project charter, project team formed, project goals defined, customer requirements identified, project metrics.
- **MEASURE:** Map and measure the process to understand current performance. Tools: Value Stream Map, Swim Lane Map, Control Charts, Histograms. Deliverables: Current state map, data collected and validated, baseline performance measured.
- **ANALYZE:** Identify amounts and types of waste; determine root causes. Tools: Fishbone Diagram, Pareto Chart, Root Cause Analysis (5 Whys), FMEA. Deliverables: Waste and issues identified, root causes analyzed, critical factors identified.
- **IMPROVE:** Design the future state and test improvements. Tools: Future State VSM, Kaizen Events, PDSA, Mistake Proofing, Visual Controls. Deliverables: Future state designed, solutions tested and validated, action plans created, pilot conducted.
- **CONTROL:** Create a process control strategy to sustain improvements. Tools: Standard Work Instructions, Control Charts, Communication Plan. Deliverables: Control system in place, communication plan implemented, full-scale implementation.

The workbook notes that "all projects do not require the use of every tool"—scope and complexity determine which tools are selected.

## Why It Matters

DMAIC provides a systematic pathway from problem identification to sustainable improvement, preventing the common failure of jumping to solutions before understanding root causes. The explicit separation of measurement from analysis forces data-driven decision-making; the control phase embeds sustainability. Because each phase has named tools, teams know what question they're answering and which methods answer it best. This makes the methodology teachable and repeatable across organizations, regardless of industry—the DMAIC logic scales from a small process defect to a portfolio of complex improvement initiatives.
