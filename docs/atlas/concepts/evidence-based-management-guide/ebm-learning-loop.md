# EBM Learning Loop: Continuous Improvement Through Organizational Improvement

## Definition

The EBM Learning Loop is a five-step cyclical process for improving organizational value delivery: (1) Quantify Value—define Key Value Measures for the Key Value Areas; (2) Measure KVMs—establish baseline measurements; (3) Select KVAs to improve—prioritize which areas to target; (4) Conduct experiments—test practices hypothesized to improve the selected KVAs; (5) Evaluate results—compare post-experiment measurements to baseline, and iterate. This loop runs continuously, making small changes, measuring impact, and cycling again rather than waiting for large improvements or annual strategic reviews.

## In the Book

The Evidence-Based Management Guide presents the loop as answer to the question "How to Improve Empirically using EBM": 

"1. Quantify Value – The first step in the EBM learning loop is to quantify value in the form of KVMs. The process of defining and aligning around KVMs itself might be valuable to an organization, because it can create transparency around what is being optimized."

"2. Measure KVMs – The next step...is to establish initial values or a baseline measure for the KVMs of interest. This step provides an initial view of the viability of the product and the ability of the organization to deliver the product. It creates transparency around relative strengths and weaknesses."

"3. Select KVAs to improve – With a clear view of current organizational value...managers...can now make informed decisions about which KVAs would be most valuable to change. Don't try to affect too many KVAs within a single learning loop. Making small incremental changes and then quickly measuring the result is better than delaying improvement and measurement by changing too many factors at once."

"4. Conduct practice experiments – After choosing a desired KVA to improve, select, at most, a few practices...and run an experiment."

"5. Evaluate results – Once the results of the experiment are measured, they should be compared with the KVM values before the changes were made. If the experiment produced improved measured, the change is kept, otherwise other improvements are tried. The learning loop continues until the KVA reflect the desired results."

The guide emphasizes iteration as the core mechanism: "Improving organizational performance is also a cyclic, iterative process: measuring current conditions, setting performance goals, forming small improvement experiments that can be quickly run, and then measuring again to gauge the effect, then repeating, continuously."

## Why It Matters

Most organizations improve sporadically: a crisis drives change, or an annual strategy review identifies fixes. The EBM Learning Loop makes improvement continuous and empirical. By running many small experiments with rapid feedback, teams avoid the trap of large changes that produce unmeasurable impact. The loop also forces clarity about *what* to improve (it surfaces disagreements when teams choose which KVA to target). In any complex adaptive system, continuous small iterations with measurement beat episodic large changes with uncertain impact. The loop's power lies in its discipline: you only proceed to experiment if you've measured baseline, and you only keep changes if you've measured impact.
