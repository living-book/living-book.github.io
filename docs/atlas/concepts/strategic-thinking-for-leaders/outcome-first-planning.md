# Outcome-First Planning (The ABCs of Systems Thinking)

## Definition

Haines's "ABCs" are five questions asked in a fixed order on any planning problem,
from a single project to an entire organization's strategy: A — where do we want
to be (vision)? B — how will we know we got there (feedback)? C — where are we now
(current state)? D — how do we get there (strategies and actions)? E — what is
changing in the environment (ongoing)? The load-bearing move is the ordering: vision
is defined and locked in *before* the current state is assessed, reversing the
instinct to start every plan by cataloguing today's problems.

## In the Book

Chapter II sets this explicitly against "Analytic Thinking," which the book defines
as starting with today's issues, breaking them into their smallest components, and
solving each piece separately with "no far-reaching vision or goal — just the
absence of the problem." The ABC sequence inverts that: "In Systems Thinking, the
whole is primary and the parts are secondary." Tip #3 ("Define Your Harbor") makes
the case with Seneca's line — "when a man does not know what harbor he is making
for, no wind is the right wind" — and argues that without a shared destination
defined first, employees' individually reasonable decisions drift in slightly
different directions, and the accumulated drift wastes time, money, and energy.
Only after the harbor (Phase A) is set does Tip #5 turn to the SWOT-based Current
State Assessment (Phase C) — "prior to making any changes... you need to determine
two key points of change: where you are going... [and] where you are now" — in that
order.

## Why It Matters

Diagnosing a problem before defining what success looks like tends to anchor the
plan on removing what's currently wrong rather than on reaching a positive target —
you end up optimizing symptoms instead of designing toward an outcome. Naming the
destination first gives everyone downstream a shared reference point against which
"is this action taking us there" can be asked directly, and it converts current-state
diagnosis from an open-ended problem hunt into a bounded gap analysis (distance from
C to A) — useful anywhere a team's first instinct is to start solving before anyone
has agreed on solving toward what.
